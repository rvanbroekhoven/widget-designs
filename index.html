Developer Handoff: Volledige Specificaties

Dit referentiekader omvat tot in het kleinste detail alle actuele ontwerpkeuzes, CSS variabelen, afmetingen en interactieregels ter ondersteuning van een pixel-perfect development build.

1. Kleurpalet & Variabelen

  • Navy (Primary): var(--vrmwb-navy) / #122053. Gebruikt voor topbars, opslaan-knoppen, kalenderheaders, en hover states van foto-modal controls.
  • Red (Alert): var(--vrmwb-red) / #D2100B. Gebruikt voor delete-knoppen, sluit-iconen (modals), afgevinkte werklijsten en sticky table headers (via shadow).
  • Gold (Accent): var(--vrmwb-gold) / #CC9933. Gebruikt voor widgettitels, hover-states op Navy elementen, edit-knoppen en actieve tab-borders.
  • Tekst Main: var(--text-main) / #1E1E1E. Standaard tekstkleur.
  • Tekst Muted: var(--text-muted) / #707172. Voor timestamps, secundaire info en placeholders.
  • Input / Inner BG: var(--input-bg) / rgba(0,0,0,0.04) met border var(--input-border) / #E5E5E5.

2. Typografie & Styling

  • Font Family: 'Open Sans', sans-serif. Altijd forceren via CSS.
  • Base styling: Base font-size is 14px met een line-height van 1.6. Paragrafen in Tekst Widgets gebruiken margin-bottom: 10px.
  • Hoofdkoppen (Widget Titels): Grootte 18px, gewicht 900 (Black), kleur Gold, text-transform: uppercase, en letter-spacing 0.5px.
  • Subtekst & Metadata (w-meta-row): Grootte 11px, kleur Muted, font-style: italic. Staan onderaan in widgets, gescheiden door een dashed border 1px dashed var(--input-border).
  • Truncation (Afkorten): Lange teksten in tabellen en werklijsten worden hard afgebroken met puntjes via de class .truncate-table-text. white-space: nowrap; overflow: hidden; text-overflow: ellipsis; max-width: 170px (Size S) / 300px (Size L);

3. Layout, Grid & Spacing

  • Body Padding: 25px rondom de viewport om de schermruimte horizontaal te maximaliseren.
  • Dashboard Grid (Container): Gebruikt display: flex; flex-wrap: wrap; gap: 15px;. De gap van 15px is essentieel om Size S, L en Modal op de meeste 1080p schermen exact op 1 rij te laten passen.
  • Widget Flex-basis: Widgets (divs direct in de dashboard-grid) hebben flex: 0 0 auto; display: flex; flex-direction: column; om hun vaste formaten te waarborgen en te voorkomen dat ze ongewenst uitrekken.
  • Widget Formaten:
    • Size S: width: 320px; height: 320px;
    • Size L: width: 670px; height: 320px;
    • Size Mobile: width: 375px; height: 667px;
  • Widget Opbouw: Een widget bestaat uit .w-header, evt. .w-tab-container, en .w-body. De body heeft flex: 1; overflow-y: auto; zodat content scrollbaar is maar de header en footer blijven staan.

4. Schaduwen, Randen & Transities

  • Border Radii: Buitenranden van widgets (var(--radius-outer)) zijn 20px. Binnen-elementen zoals foto's, inputs en tabs (var(--radius-inner)) gebruiken 8px of 12px.
  • Base Shadow: Widgets en UI-containers hebben standaard box-shadow: 0 8px 30px rgba(0,0,0,0.08), 0 2px 8px rgba(0,0,0,0.04).
  • Hover States: Vrijwel alle interactieve elementen (widgets, locatieknoppen, galerijknoppen) hebben: transition: transform 0.2s, box-shadow 0.2s;
    transform: translateY(-2px);
    box-shadow: 0 12px 40px rgba(0,0,0,0.12);
  • Tab borders: Actieve tabs krijgen position: relative; top: 2px; om hun bottom-border naadloos over de container-lijn te laten vallen.

5. Datepicker (Custom Flatpickr)

  • Header Injectie: Standaard header verbergen met .flatpickr-months { display: none !important; }. Een eigen Navy div wordt via de onReady JS functie geïnjecteerd (maand/jaar gescheiden).
  • Pijltjes Centrering: Pijltjes in de header en topbar gebruiken padding: 0 5px 5px 5px om het Frutiger `‹` en `›` karakter optisch exact verticaal te centreren.
  • Grid & Zondag-Fix: Om te voorkomen dat de rechterkant (zondag) buiten het paneel valt of geen padding heeft: .flatpickr-weekdaycontainer, .dayContainer {
    display: grid !important;
    grid-template-columns: repeat(7, 1fr) !important;
    width: 100% !important; box-sizing: border-box;
    }

6. Locatie & Piket Dropdown

  • Responsive Breedte: Geen vaste width. Gebruik white-space: nowrap op .loc-item. De dropdown (min-width: 220px) schaalt vanzelf mee met extreem lange kazernenamen (zoals 'OVD - Hart van Brabant').
  • Piket (Nested): Het Piket-menu is opgebouwd in een .loc-nested-wrap. De chevron toggle draait 180 graden op open: transform: rotate(180deg).
  • Visueel onderscheid: Het uitgeklapte piket menu heeft background: rgba(0,0,0,0.04) en margin-top 4px om visueel los te staan van de hoofd-kazernes.

7. Foto Galerij (Full Screen)

  • Galerij Trigger: Voeg aan thumbnail-knoppen class open-gallery-btn toe en de attributenlijst data-gallery='["img/1.jpg", "img/2.jpg"]'. De JS leest dit als array.
  • Navigatie Logica: JS gebruikt Modulo (%) voor een infinite loop. Bij Volgende op de laatste foto index: (index + 1) % array.length. Bij Vorige: (index - 1 + array.length) % array.length.
  • Vormgeving Navigatie: Ronde knoppen (60x60px), background: var(--vrmwb-navy), color: #FFF. Hover: Goud en transform: scale(1.05).
  • Afbeelding Placeholder: Indien een afbeelding traag laadt of breekt, is het #modal-img element gestyled als grijs vlak met min-width: 60vw; min-height: 60vh; background-color: var(--input-bg); color: transparent;.

8. Z-Index & Browser Fixes

  • Z-Index Structuur: Om overlapping bugs in complexe dashboards te voorkomen gelden harde regels:
    • z-index: 100 -> Topbar
    • z-index: 300 -> Reguliere dropdowns
    • z-index: 9999 -> Locatie Dropdown (`.loc-menu` en container `.loc-wrap.open`). Dit forceert de dropdown over micro-actie kruisjes heen.
    • z-index: 100000+ -> Modals & Fullscreen overlays.
  • Safari Sticky Table Header Fix: Een border valt in Safari vaak weg onder scrollende elementen in een sticky tr. Oplossing: gebruik een box-shadow in plaats van border-bottom op de table headers: th { position: sticky; top: -1px; box-shadow: inset 0 -2px 0 var(--vrmwb-red); }
