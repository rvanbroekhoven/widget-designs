Developer Handoff: Uitgebreide Specificaties

Dit referentiekader omvat de harde ontwerpkeuzes, afmetingen en interactieregels ter ondersteuning van de development build in ODj Widget en ODj Beheer.

Kleurcodes & Theming

  • Navy (Primary): #122053 - Voor topbars, primaire opslaan-knoppen en headers.
  • Red (Action/Alert): #D2100B - Voor Delete-knoppen, afgevinkte werklijsten en table-headers.
  • Gold (Accent): #CC9933 - Voor Hoofdtitels van de widgets, Edit-knoppen en actieve tab-lijnen.
  • Text Main: #1E1E1E / Text Muted: #707172
  • Widget BG: #FFFFFF (met Card Border rgba(0,0,0,0.08)).

Typografie & Structuur

  • Font-Family: Uitsluitend 'Frutiger', sans-serif.
  • Font Weights: 300 (Pijltjes/Light), 400 (Regulier/Paragraph), 700 (Knoppen, Labels), 900 (Hoofdtitels).
  • Header Structuur: De Hoofdtitel (`.w-title`) is goud (`#CC9933`), font-weight 900. Rechterkant bevat `.w-controls` voor bewerken/verwijderen van de widget op het dashboard.
  • Native Rendering: Font-smoothing is uitsluitend via antialiased ingesteld om overal het originele sub-pixel karakter van de Frutiger font the behouden.

Architectuur & Afmetingen

  • Size Small (S): Hard vastgezet op 320px breed, 320px hoog.
  • Size Large (L): Hard vastgezet op 670px breed, 320px hoog.
  • Size Modal: Hard vastgezet op 375px breed, 667px hoog. Gebruikt een Navy basis: rgba(18, 32, 83, 0.95) met backdrop-filter: blur(10px).
  • Border-radius: Buitenkant widget is 20px, binnen-elementen (knoppen, inputs) 12px of 8px.
  • Schaduw: Standaard widget drop-shadow is 0 8px 30px rgba(0,0,0,0.08).

Widget Logica (Apple Filosofie)

  • Doel: De content-dichtheid schaalt mee, maar de data is áltijd synchroon. De weergave is puur "View-Only".
  • Small Logic: Gebruik in tabellen alleen Tijd + Melding. In werklijsten, kap de lange tekst af met white-space: nowrap; text-overflow: ellipsis;. In tekst-widgets worden de footer metadata (Aangemaakt, Bewerkt) gestapeld.
  • Large/Modal Logic: Tabellen tonen alle 4 kolommen (L) of 2 brede kolommen (Modal). Werklijsten tonen de metadata "Toegevoegd door" & "Toegewezen aan" direct onder de content. Tekst-widget metadata (`.w-meta-row`) staat in L op 1 vloeiende horizontale regel.
  • Voetnoot (Werklijsten): Werklijsten hebben altijd een dynamische 'Nog X taken open' footer die het aantal niet-afgevinkte taken berekent. Klikken hierop opent het overzicht (Modal) met resterende taken. Indien alle taken afgevinkt zijn, toont deze "Geen taken open".
  • Foto's: Het Tekstblok toont altijd de bijbehorende foto's (als tag-knop in S, als grid in L/Modal).

Interactie & Cursor Gedrag

  • Cursor States: Pas altijd cursor: pointer; toe op alle klikbare elementen (knoppen, tabbladen, checkboxes, foto-thumbs, tabel-rijen en gehele werklijst-rijen) zodat de interactie duidelijk is.
  • Lift Animatie (Hover): Elementen hebben een tactiele 'lift' op hover: transform: translateY(-2px); gekoppeld met box-shadow: var(--shadow-hover);.
  • Soepele Transities: Alle hover-statussen moeten vloeiend verlopen. Gebruik hiervoor standaard transition: all 0.2s ease;.
  • Modals Activeren: Klikken op een foto opent de Full-Screen Foto Modal. Klikken op een specifieke taak (of melding in tabel) opent de desbetreffende Detail Modal. Let op: de sluit-kruisjes in modals delen exact de styling van de overige .icon-btn en .modal-close-btn knoppen.

Cross-Browser & Rendering Fixes

  • Safari Tabel-spleet: Webkit heeft een sub-pixel bug bij `position: sticky`. Fix dit door de `<th>` op `top: -1px;` te zetten met een `border-top` als onzichtbaar dakje. Teken de rode lijn via `box-shadow: inset 0 -2px 0 var(--vrmwb-red);`.
  • Dashboard Layout: Om te voorkomen dat widgets in Chrome naar een nieuwe regel wrappen, gebruikt de `.dashboard-grid` flex-wrap: nowrap; met overflow-x: auto;.
  • Font Rendering: Gebruik uitsluitend -webkit-font-smoothing: antialiased; op de body zodat de Frutiger font in alle browsers (Safari én Chrome) dezelfde strakke weergave behoudt, zónder de originele dikte te verliezen.
  • Tab Wiebelen: Tab-containers hebben expliciet overflow-y: hidden; gekregen zodat de horizontale scrollbar niet zorgt voor een verticale 'wiebel'.
